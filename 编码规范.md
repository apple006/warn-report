## **编码规范**
### **RestfulAPI URL命名规范**
1. 基础规范(参见RESTful web APIs)
2. http中的几个方法:GET,POST,PUT,DELETE
例如:
GET /orders/{orderId} HTTP/1.1
此请求为获取订单下的一个订单信息.因为订单不只是一个,所以我们将一级URL定义为orders当然
GET /orders/ HTTP/1.1
此请求就是获取订单列表了.

### **通用头部的定义**

名称 | 类型 | 作用 | 值 | 所在区域
--- | --- | --- | --- | ---
Connection | String | 连接类型 | keep-alive                      | RequestHeader
Cache-Control | String | 文档新鲜度 | no-cache                         | RequestHeader
Accept | String | 客户端接受的媒体类型 | 按照需求自定义                  | RequestHeader
Accept-Encoding | String | 媒体类型的编码方式(例如主流的Gzip压缩方式,此处告诉服务器,客户端支持的编码方式) | gzip, deflate, sdch | RequestHeader
Accept-Language | String | 媒体类型的语言喜好度(可用此头部做国际化) | 按需求自定义 | RequestHeader
Host | String | 主域名 | 发送请求所到服务器的主机名,对相对 URL的计算有相当大的作用 | RequestHeader
User-agent | String | 代理用户发送请求的浏览器或者telnet客户端自动添加的头部,主要解决浏览器兼容问题 | 浏览器或者telnet客户端自带 | RequestHeader
If-Modified-Since | Date | 发送条件GET的条件 | 来自上次响应头Last-Modified的值 | RequestHeader
If-None-Match | String | 发送条件GET的条件 | 来自上次响应头的ETag的值 | RequestHeader
Age | String | 响应时间的记录,可以调试应用程序是追踪请求的时间爱你是否合理 | 服务器自己记录 | ResponseHeader
Server | String | 记录响应来自那个服务器(即经过的最后一个服务器的服务器唯一标识) | 服务器自己记录 | ResponseHeader
Vary | String | Accept-Encoding | 未知 | ResponseHeader
Content-Encoding | String | 响应报文的编码方式 | 来自Accept-Encoding | ResponseHeader
Content-Length | String | 响应报文的长度(只有在报文存在时会存在此报头) | 服务器自动识别报文长度 | ResponseHeader
Content-Language | String | 响应报文的语言 | 来自Accept-Language | ResponseHeader
Content-Type | String | 响应报文的类型 | 自定义(控制报文传输时的类型.在本次2.0中,全部为json) | ResponseHeader
Etag | String | 实体标签(用于发送条件get,与httpcache密切相关) | 自定义(通过编程实现) | ResponseHeader
Cache-Control | String | 控制文档新鲜度(与httpcache密切相关) | max-age:自定义 | ResponseHeader
Via | String | 记录请求路过的服务器(每当响应经过一个节点服务器,服务器会在这个首部加上自己唯一id作为标识可以通过此报头了解响应经过的路径) | 服务器自己记录 | ResponseHeader
Date | String | Request请求的时间 | 上次请求的发出时间               | ResponseHeader

### **SQL注入防御安全实践**

1. 当传入的参数是单个的值时：
- 应当先使用 ? 占位符拼接sql
- 再将需要传入的值在调用dao方法时候传入
2. 当传入的参数为String,且格式为1,2,3,4时：
- 先String分解为字符串数组
- 在用 ? 占位符循环拼接sql并且将值设置到需要传入的dao方法的集合中
- 最后调用dao方法执行sql
3. 当使用模糊查询时：
- 将传入的值,重构
- 用?占位符,拼接SQL
- 调用方法执行
4. 当传入的是数组时：
- 循环使用占位符 ? 拼接SQL
- 调用dao方法.并直接将数组传入其中

### **代码注释实践**
```
@author   对类的说明 标明开发该类模块的作者
@version  对类的说明 标明该类模块的版本
@see     对类、属性、方法的说明 参考转向，也就是相关主题
@param对方法的说明 对方法中某参数的说明
@return   对方法的说明 对方法返回值的说明
@throws 抛出方法中的异常

```

#### 常用注释方式
##### 类（接口）注释
```
/**
 * <p>描述内容</br>${todo} 简要描述,描述结束请删除TODO</p>
 * @author ${user} Email:${user}@caijinquan.com
 * @company 财金圈（北京）金融服务外包有限公司@版权所有
 * @version  
 * @since ${date}${time}
 */

```

##### 方法注释
```
/**
 * <p>必填项：方法简要描述</p>
 * <p>简要描述该方法的作用以及应用场景</br>其他参数根据实际情况进行处理</p>
 * ${tags}
 */

```
方法注释中如果遇到多个参数，需要用@param 将每个参数的作用进行说明。如果遇到map类型的参数，需要将map 的key对应的参数进行说明，如果遇到boolean类型的参数，需要说明true代表什么，false代表什么，如果遇到分页包装类，需要将类的信息进行说明。

##### 全局变量注释
```
/** ABC */
private static final String ABC = "ABC";
```

##### 字段属性注释
```
private String id;// 唯一id
private String userName;// 用户名
```
